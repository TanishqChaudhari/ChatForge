SOCKET.IO EVENT FLOW
═══════════════════════════════════════════════════════════════════════════════

CONNECTION:
  Client connects with JWT token in auth
    ↓
  Server middleware verifies token
    ↓
  Extract userId from token
    ↓
  User joins their personal room: io.to(userId)
    ↓
  Broadcast: user_online event
    ↓
  Connection complete

SENDING MESSAGE:
  Client: socket.emit('send_message', {conversationId, content})
    ↓
  Server receives event
    ↓
  Verify user is conversation participant
    ↓
  Save message to MongoDB
    ↓
  Update conversation.lastMessage
    ↓
  Send confirmation: socket.emit('message_sent', {messageId, status})
    ↓
  Send to receiver: io.to(receiverId).emit('message_received', {message})
    ↓
  Receiver receives in real-time!

MESSAGE READ RECEIPT:
  Receiver: socket.emit('message_read', {messageId, conversationId})
    ↓
  Server finds message and updates status to "read"
    ↓
  Record readAt timestamp
    ↓
  Send receipt to sender: io.to(senderId).emit('message_read_receipt')
    ↓
  Sender's app shows: ✓✓✓ (read)

TYPING INDICATOR:
  User starts typing: socket.emit('typing_start', {conversationId})
    ↓
  Server broadcasts: io.to(conversationId).emit('user_typing', {user})
    ↓
  All participants see: "Alice is typing..."
    ↓
  User stops typing: socket.emit('typing_stop', {conversationId})
    ↓
  Server broadcasts: io.to(conversationId).emit('user_stopped_typing')
    ↓
  UI updates: "typing" indicator disappears

HOW TO USE PHASE 3
═══════════════════════════════════════════════════════════════════════════════

METHOD 1: REST API (HTTP)
  
  Send Message:
  POST /api/messages
  Headers: Authorization: Bearer {token}
  Body: {"conversationId": "...", "content": "Hello!"}
  
  Get Messages:
  GET /api/conversations/{convId}/messages?page=1&limit=20
  Headers: Authorization: Bearer {token}
  
  Mark as Read:
  PATCH /api/messages/{messageId}/read
  Headers: Authorization: Bearer {token}
  
  Update Status:
  PATCH /api/messages/{messageId}/status
  Headers: Authorization: Bearer {token}
  Body: {"status": "delivered"}


METHOD 2: Socket.io (Real-time WebSocket)

  Connect with Auth:
  ```javascript
  const socket = io('http://localhost:8000', {
    auth: { token: jwtToken }
  });
  ```
  
  Send Message:
  ```javascript
  socket.emit('send_message', {
    conversationId: convId,
    content: 'Hello Bob!'
  });
  ```
  
  Listen for Messages:
  ```javascript
  socket.on('message_received', (message) => {
    console.log('New message:', message.content);
  });
  ```
  
  Typing Indicator:
  ```javascript
  socket.emit('typing_start', { conversationId: convId });
  socket.emit('typing_stop', { conversationId: convId });
  ```
  
  Read Receipt:
  ```javascript
  socket.emit('message_read', {
    messageId: msgId,
    conversationId: convId
  });
  ```


ROUTING & ROOM MANAGEMENT
═══════════════════════════════════════════════════════════════════════════════

User Room:
  - Each user joins room with their userId on connect
  - Private messages sent to user's room
  - Example: io.to(userId).emit('message_received', message)

Conversation Room:
  - Optional: Users can join conversation room explicitly
  - Typing indicators broadcast to conversation room
  - Example: io.to(conversationId).emit('user_typing', {user})

Automatic Rooms:
  - Socket automatically in its own socket.id room
  - Socket automatically in userId room (user's personal room)
  - Can explicitly join conversation rooms

ERROR HANDLING
═══════════════════════════════════════════════════════════════════════════════

Socket.io Errors:
  ├── "Authentication required" → No token provided
  ├── "Invalid token" → Token verification failed
  ├── {message: "..."} → Event processing errors sent via socket.emit('error')
  └── All operations wrapped in try/catch

REST API Errors:
  ├── 400 Bad Request → Missing required fields
  ├── 401 Unauthorized → No/invalid token
  ├── 403 Forbidden → Not participant in conversation
  ├── 404 Not Found → Resource doesn't exist
  └── 500 Internal Server Error → Database/server errors
